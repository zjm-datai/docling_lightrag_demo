# 实现 SRA RAG 模块

## 概述

根据架构设计实现 `sra_rag` 模块的完整流程：文献解析 → Layout-aware 分块 → LightRAG 索引 → 向量/图检索。

**技术栈：**
- Docling：文档解析（PDF/DOCX/XLSX → DoclingDocument）
- LightRAG：双级检索（向量 + 知识图谱）
- LLM：Qwen3-30B-A3B-GPTQ-Int4（OpenAI 兼容 API）
- Embedding：bge-m3
- 存储：LightRAG 默认文件存储

---

## 实现步骤

### 1. 实现 `sra_rag/parser/base.py` - 解析器抽象基类

定义文档解析器的通用接口：

```python
from abc import ABC, abstractmethod
from pathlib import Path
from typing import Any

@dataclass
class ParsedDocument:
    """解析后的文档表示"""
    title: str
    content: str  # 结构化文本（Markdown）
    metadata: dict[str, Any]  # 标题、摘要、年份、关键词等
    chunks: list[dict]  # 分块列表，包含 text, section, page, block 等元数据

class BaseParser(ABC):
    @abstractmethod
    def parse(self, file_path: Path) -> ParsedDocument:
        pass
```

### 2. 实现 `sra_rag/parser/docling_parser.py` - Docling 解析器

使用 Docling 解析多格式文档，实现 layout-aware 分块：

**核心实现：**
- 使用 `DocumentConverter` 解析 PDF/DOCX/XLSX
- 从 `DoclingDocument` 提取结构化内容
- 按文档层级（section/page/block）进行智能分块
- 保留元数据：section title, page number, block type

```python
from docling.document_converter import DocumentConverter
from docling_core.types.doc import DoclingDocument

class DoclingParser(BaseParser):
    def __init__(self, allowed_formats: list | None = None):
        self.converter = DocumentConverter(allowed_formats=allowed_formats)
    
    def parse(self, file_path: Path) -> ParsedDocument:
        result = self.converter.convert(file_path)
        doc = result.document
        
        # 提取元数据
        metadata = self._extract_metadata(doc)
        
        # 导出为 Markdown
        content = doc.export_to_markdown()
        
        # Layout-aware chunking
        chunks = self._chunk_by_layout(doc)
        
        return ParsedDocument(
            title=metadata.get("title", file_path.stem),
            content=content,
            metadata=metadata,
            chunks=chunks
        )
    
    def _chunk_by_layout(self, doc: DoclingDocument) -> list[dict]:
        """按文档结构分块，保留 section/page/block 元数据"""
        chunks = []
        current_section = ""
        
        for item in doc.texts:
            # 识别章节标题
            if hasattr(item, 'label') and item.label in ['section_header', 'title']:
                current_section = item.text
            
            # 获取页面信息（如果有 layout 数据）
            page_num = self._get_page_number(item)
            
            chunks.append({
                "text": item.text,
                "section": current_section,
                "page": page_num,
                "block_type": getattr(item, 'label', 'paragraph'),
                "metadata": {
                    "title": doc.name,
                    "section_path": self._get_section_path(item)
                }
            })
        
        return chunks
```

**关键点：**
- 遍历 `doc.texts` 获取所有文本项
- 识别 `section_header`、`title` 等标签维护层级
- 从 `item.prov` 提取页面信息（bounding box）
- 表格、图片单独处理

### 4. 实现 `sra_rag/indexer/lightrag_indexer.py` - LightRAG 索引器

集成 LightRAG，配置 Qwen3 和 bge-m3：

```python
from lightrag import LightRAG, QueryParam
from lightrag.llm.openai import openai_complete_if_cache
from lightrag.utils import EmbeddingFunc

class LightRAGIndexer:
    def __init__(
        self,
        working_dir: str = "./sra_rag_data",
        llm_base_url: str = "http://211.90.240.240:30001/v1",
        llm_api_key: str = "gpustack_ba7863cefc4126b2_4ecbb5f42c239594a33e67ef49bdce9d",
        llm_model: str = "Qwen3-30B-A3B-GPTQ-Int4",
        embedding_model: str = "bge-m3",
        embedding_dim: int = 1024,
    ):
        self.working_dir = working_dir
        os.makedirs(working_dir, exist_ok=True)
        
        # 配置 LLM 函数
        llm_func = partial(
            openai_complete_if_cache,
            model=llm_model,
            base_url=llm_base_url,
            api_key=llm_api_key,
        )
        
        # 配置 Embedding 函数
        embedding_func = EmbeddingFunc(
            embedding_dim=embedding_dim,
            max_token_size=8192,
            func=lambda texts: self._embed_texts(texts, embedding_model, llm_base_url, llm_api_key)
        )
        
        # 初始化 LightRAG
        self.rag = LightRAG(
            working_dir=working_dir,
            llm_model_func=llm_func,
            embedding_func=embedding_func,
        )
    
    def index_document(self, doc: ParsedDocument) -> str:
        """索引解析后的文档"""
        # 使用 chunks 进行索引
        chunk_texts = [chunk["text"] for chunk in doc.chunks]
        
        # 插入到 LightRAG（会自动构建知识图谱）
        self.rag.insert(chunk_texts)
        
        return doc.title
    
    def query(
        self,
        query: str,
        mode: str = "hybrid",
        top_k: int = 5
    ) -> dict:
        """查询检索"""
        param = QueryParam(mode=mode)
        result = self.rag.query(query, param=param)
        
        return {
            "answer": result,
            "mode": mode
        }
    
    async def _embed_texts(self, texts: list[str], model: str, base_url: str, api_key: str):
        """调用 bge-m3 嵌入模型"""
        # 使用 OpenAI 兼容接口调用 bge-m3
        import aiohttp
        
        async with aiohttp.ClientSession() as session:
            response = await session.post(
                f"{base_url}/embeddings",
                headers={"Authorization": f"Bearer {api_key}"},
                json={"model": model, "input": texts}
            )
            data = await response.json()
            
            embeddings = [item["embedding"] for item in data["data"]]
            return embeddings
```

**关键点：**
- `working_dir` 存储所有索引数据（向量、图、元数据）
- 使用 `openai_complete_if_cache` 调用 Qwen3
- 自定义 `_embed_texts` 调用 bge-m3
- `insert()` 自动处理分块、实体提取、关系构建

### 6. 实现 `sra_rag/retrieval/lightrag_retrieval.py` - LightRAG 检索器

封装 LightRAG 的多种检索模式：

```python
class LightRAGRetriever:
    def __init__(self, indexer: LightRAGIndexer):
        self.rag = indexer.rag
    
    def retrieve(
        self,
        query: str,
        mode: str = "hybrid",
        top_k: int = 5
    ) -> list[RetrievalResult]:
        """
        检索模式：
        - naive: 简单向量检索
        - local: 局部知识图谱检索
        - global: 全局知识图谱检索
        - hybrid: 混合检索（推荐）
        """
        param = QueryParam(mode=mode)
        result = self.rag.query(query, param=param)
        
        return [
            RetrievalResult(
                content=result,
                metadata={"mode": mode},
                score=1.0
            )
        ]
```

## 数据流示例

```python
# 完整使用示例
from pathlib import Path
from sra_rag import DoclingParser, LightRAGIndexer, LightRAGRetriever

# 1. 解析文档
parser = DoclingParser()
parsed_doc = parser.parse(Path("paper.pdf"))

# 2. 索引
indexer = LightRAGIndexer(working_dir="./rag_data")
doc_id = indexer.index_document(parsed_doc)

# 3. 检索
retriever = LightRAGRetriever(indexer)
results = retriever.retrieve("什么是图神经网络？", mode="hybrid")
```

## 注意事项

1. **LightRAG 版本兼容性**：使用 `lightrag-hku>=1.4.16`，API 可能有变化，需要适配
2. **bge-m3 嵌入维度**：bge-m3 的 embedding_dim 是 1024（不是 768）
3. **异步 Embedding**：LightRAG 索引过程是异步的，需要正确处理 async/await
4. **首次索引时间**：LightRAG 需要提取实体和构建知识图谱，首次索引较慢
5. **存储结构**：`working_dir` 会生成多个文件（kv_store*.json, graph*.graphml），需要纳入 .gitignore
6. **Chunk 元数据传递**：当前 LightRAG 的 `insert()` 接受纯文本列表，元数据需要在外层维护映射关系
