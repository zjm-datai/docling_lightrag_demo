
## 一、前端展示：输入表单设计

用户需要填写的输入项，按区块划分：

### 区块1：研究基本信息（文本+选项）

| 字段名     | 组件类型 | 必填  | 说明                         |
| ------- | ---- | --- | -------------------------- |
| 研究目标    | 文本域  | 是   | 例如：研究疾病（特应性皮炎）、研究人群（儿童/成人） |
| 研究方向/热点 | 文本输入 | 否   | 例如：肠皮轴、炎症通路、Th2 免疫调控       |
| 结局指标    | 文本输入 | 否   | 例如：EASI、瘙痒评分、IL-4、IgE      |
| 创新点     | 多选框  | 否   | 理论创新、方法创新、应用创新（可多选）        |
| 研究类型    | 单选   | 是   | 药理机制、方剂组合规律                |
| 输出用途    | 单选   | 是   | 开题报告、基金申报、立项论证、论文选题、科研储备   |

### 区块2：文献输入（支持两种方式）

| 方式     | 组件          | 说明                                           |
| ------ | ----------- | -------------------------------------------- |
| 系统推荐文献 | 列表多选        | 系统预先检索的文献，用户勾选                               |
| 用户自选文献 | 文件上传 + 文献列表 | 支持 PDF / Word / Excel，自动解析；也可手动输入标题/DOI/PMID |

> 说明：允许**单篇或多篇**。用户至少选择 1 篇文献。

### 区块 3：高级选项（可选）

| 字段 | 组件 | 说明 |
|------|------|------|
| 重点关注机制/通路 | 文本输入 | 用于药理机制分析时优先聚焦 |
| 重点关注药物/方剂 | 文本输入 | 用于方剂组合规律分析 |
| 是否包含安全性/经济性分析 | 开关 | 默认开启 |

## 二、后端智能体协作框架

设想的每个模块作为一个**独立智能体（Agent）**，各智能体共享：

- 用户输入的原始研究内容
- 用户选择的文献（已解析 + 向量化索引）
- 系统文献库（预置文献集）

每个智能体执行步骤：

1. **检索**：根据自身需求，向向量库发起检索（如：查询与 “EASI评分” 相关的文献段落）
2. **组装上下文**：用户输入 + 检索结果 + 系统提示词
3. **生成**：输出对应模块内容

---

## 三、每个输出模块所需的具体内容

以下是每个智能体输出时需要的**核心内容**与**检索策略**，您可用此与产品对齐。

### 1. 研究内容概述

**输出目标**：用自然语言概括用户拟研究的问题、对象与方向。

**需要的上下文**：

- 用户填写的所有研究基本信息（研究目标、人群、方向、指标、创新点、研究类型、用途）

**检索需求**：

- 不需要检索文献

**输出示例**：

> 本研究拟以特应性皮炎（AD）儿童人群为对象，围绕 “肠皮轴” 调控机制，评估某方剂对 EASI 评分及 IL-4 水平的影响，属于药理机制研究，预期用于基金申报。

---

### 2. 研究现状总结

**输出目标**：基于文献，归纳：

- 已被充分研究的内容
- 正在升温的热点内容
- 证据较少但有潜力的内容
- 不同研究之间的分歧点

**需要的上下文**：

通过用户输入的研究目标、方向、热点做拆分关键词去得到文献库中的关联文献，取得文献库中文献的元数据（标题、摘要、发表年份、关键词等信息）；

通过这些信息作为上下文去让大模型生成输出目标。

---

### 3. 创新性分析

**输出目标**：分别评估理论创新、方法创新、应用创新。每项包含：

- 已有研究
- 创新程度分析
- 可行性分析

**需要的上下文**：

检索创新点，可能在摘要里面也可能在正文，将检索到的内容给到大模型

---

### 4. 可研究点及科研思路

**输出目标**：根据研究类型和用途，生成若干个 “具体可研究点”，并为每个点匹配科研思路。

**需要的上下文**：

- 用户输入（疾病、方向、指标、研究类型、用途）
- 研究现状中的“潜力内容”和“分歧点”
- 文献中的空白/不足

---

### 5. 安全性研究

**输出目标**：评估该研究方向中潜在的药物/方剂安全性问题。

**需要的上下文**：

- 用户输入的药物/方剂（若有）
- 文献中的不良反应、毒性、禁忌信息

如果用户输入了特定的药物，需要拿这个药物去文献中检索关联和该药物有关的片段

如果没有输入药物而是输入一个病之类的就查和这个病有关的所有药物（未来后续可能需要知识库介入

---

### 6. 经济性研究

**输出目标**：分析研究方向的成本效益或经济价值。

**需要的上下文**：

- 疾病负担（如 AD 治疗成本）
- 干预措施的成本（如中药 vs 生物制剂）

**检索策略**：

- query：“特应性皮炎 + 经济学评价 + 成本”
- 若无，则建议开展药物经济学评价。

---

### 7. 当前研究热点结合建议

**输出目标**：结合前沿热点（如多组学、AI、真实世界研究），给出加分建议。

**需要的上下文**：

- 用户的研究方向 + 研究类型
- 系统热点知识库（不限于用户文献）

**检索策略**：

- query：“特应性皮炎 + 2025 研究热点 + 前沿”
- 输出：例如“建议整合肠道宏基因组与代谢组，增强理论创新性”。

---

### 8. 结论与优先推荐方向

**输出目标**：综合以上模块，给出1-2个最值得优先开展的研究方向，输出结果就是论文标题

**需要的上下文**：

- 创新性分析结果
- 可研究点
- 安全性与经济性结论

**检索策略**：

- 无需检索，综合前面模块输出。

---

## 四、输出整体结构

最终生成的《科研创新性分析报告》按以下顺序展示（前端可折叠或分页）：

1. 研究内容概述
2. 研究现状总结
   - 充分研究
   - 热点内容
   - 潜力内容
   - 分歧点
3. 创新性分析
   - 理论创新
   - 方法创新
   - 应用创新
4. 可研究点及科研思路（列表，每条包含研究问题+思路）
5. 安全性研究
6. 经济性研究
7. 当前研究热点结合建议
8. 结论与优先推荐方向
